# FinTrack API — Backend Spec

## 1. Objective

Stand up the Go API server that backs the FinTrack frontend (`finance-tracker-kim`).
The frontend is UI-complete and mock-data-driven; this backend replaces those mocks with a
real database and real auth. The API must be production-shaped from day one — clean
architecture, secure, testable, horizontally scalable.

**This spec covers only the scaffold task** (database, migrations, health check, folder
structure, Docker, env, CLAUDE.md, README). Domain endpoints (transactions, budgets, etc.)
come after the scaffold is verified.

---

## 2. Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| Language | Go 1.22+ | |
| HTTP framework | Gin | Fast, middleware ecosystem |
| ORM | GORM v2 | Rapid model iteration, auto-migrate for dev |
| Migrations | golang-migrate (SQL files) | Explicit, reversible, CI-safe |
| Database | PostgreSQL 16 | |
| Auth | JWT access tokens (15 min) + stateful refresh tokens in DB | Enables multi-session, logout-all, OAuth2 |
| OAuth providers | Google, Facebook (OAuth2 PKCE flow) | Planned; scaffold auth domain now |
| Config | Viper | `.env` file + env vars + defaults, auto-mapped to typed struct |
| API Docs | swaggo/swag (Swagger 2.0) | Generated from code annotations, served at `/swagger/*` | |
| Validation | `go-playground/validator` | |
| Module | `github.com/joakim/fintrack-api` | |

**Rules:**
- `internal/domain` has zero external imports (no GORM, no Gin). Pure Go structs and interfaces.
- `internal/service` depends on domain interfaces, never on concrete repos.
- `internal/handler` depends on service interfaces, never on repos directly.
- `pkg/` packages have no knowledge of the domain.

## 5. Environment Variables

Managed by **Viper**. Priority order (highest → lowest): environment variables → `.env` file → defaults in code.

- `.env` is used for local development only — gitignored.
- `.env.example` lists every key with a placeholder value; committed to the repo.
- In production, inject real values as OS environment variables (no `.env` file).
- Viper config keys use lowercase with dots (`db.host`); env vars map via `APP_` prefix replacement (`DB_HOST` → `db.host`).

```env
# Server
APP_ENV=development           # development | production
APP_PORT=8080

# Database
DB_HOST=localhost
DB_PORT=5432
DB_USER=fintrack
DB_PASSWORD=fintrack
DB_NAME=fintrack
DB_SSLMODE=disable            # require in production

# JWT
JWT_ACCESS_SECRET=<secret>
JWT_REFRESH_SECRET=<secret>
JWT_ACCESS_EXPIRY_MINUTES=15
JWT_REFRESH_EXPIRY_DAYS=30

# OAuth (populate when implementing OAuth endpoints)
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
FACEBOOK_CLIENT_ID=
FACEBOOK_CLIENT_SECRET=
OAUTH_REDIRECT_BASE_URL=http://localhost:3000

# Swagger
SWAGGER_ENABLED=true          # set false in production to hide docs
```

## 6. Database Schema (scaffold migrations only)

**Migration 000001 — users**
```sql
CREATE TABLE users (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email       TEXT NOT NULL UNIQUE,
  name        TEXT NOT NULL,
  avatar_url  TEXT,
  password_hash TEXT,                 -- NULL for OAuth-only accounts
  provider    TEXT NOT NULL DEFAULT 'local',  -- local | google | facebook
  provider_id TEXT,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

**Migration 000002 — sessions**
```sql
CREATE TABLE sessions (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  refresh_token TEXT NOT NULL UNIQUE,
  user_agent    TEXT,
  ip_address    TEXT,
  expires_at    TIMESTAMPTZ NOT NULL,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_sessions_user_id ON sessions(user_id);
```

## 7. API Endpoints (scaffold scope)

```
GET  /health          → 200 { status: "ok", version: "...", db: "ok" }
GET  /swagger/*any    → Swagger UI (only when SWAGGER_ENABLED=true)
```

All domain endpoints (auth, transactions, etc.) are out of scope for this scaffold task.

### Swagger annotation rules

Every handler function **must** have a swaggo annotation block. Example:

```go
// HealthCheck godoc
// @Summary      Health check
// @Description  Returns server and database status
// @Tags         system
// @Produce      json
// @Success      200  {object}  response.HealthResponse
// @Router       /health [get]
func (h *HealthHandler) HealthCheck(c *gin.Context) {
```

Run `make swagger` after any annotation change. The `docs/` output is committed to the repo so the UI works without a build step. Never hand-edit files in `docs/`.

## 8. Auth Design (for future implementation)

- **Login** → create `sessions` row, return `{ access_token, refresh_token }`
- **Refresh** → validate refresh token against DB, issue new access token
- **Logout** → delete session row by refresh token
- **Logout all** → delete all session rows for `user_id`
- **OAuth** → exchange code → upsert user → create session → same token pair

## 9. Testing Strategy

- Unit tests live alongside the code they test: `foo.go` → `foo_test.go`
- Services are tested with mock repositories (interfaces make this trivial)
- Handlers are tested with `httptest.NewRecorder` — no real DB
- Integration tests (real DB) go in `*_integration_test.go` with `//go:build integration`
- Minimum 80% coverage on `internal/service` packages

## 10. Security Checklist (scaffold)

- [ ] Passwords hashed with bcrypt cost ≥ 12
- [ ] JWT secrets loaded from env, never hardcoded
- [ ] CORS restricted to frontend origin (`APP_FRONTEND_ORIGIN`)
- [ ] `DB_SSLMODE=require` enforced when `APP_ENV=production`
- [ ] Refresh tokens are single-use (rotate on use — future)
- [ ] Rate limiting middleware stub in place (fill with redis-based limiter later)
